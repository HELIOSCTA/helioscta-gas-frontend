# Frontend Styling Preferences

## Theme Direction
- Use a dark neutral palette for app surfaces (`charcoal/slate`) instead of bright blue themes.
- Use high-contrast typography with bold white labels and section headers.
- Reserve color accents for data semantics (for example positive/negative values), not for general layout chrome.

## Cash/Hub View Defaults
- Keep explicit section headers in Historcials:
  - `Current Month`
  - `Historical Cash Prices`
  - `Forward Curve`
- Keep table headers, tab labels, and control labels bold and high contrast.
- Keep toggle buttons and selectors visually consistent across tabs (same border radius, weight, and hover behavior).

## Chart Styling
- Use dark chart backgrounds with subtle gray grid lines and axis strokes.
- Use white/near-white tick and legend text for readability.
- Keep tooltips dark with clear borders and concise numeric formatting.

## UX Constraints
- Preserve existing calculations and data logic unless a UI request explicitly requires changes.
- Preserve responsive behavior on desktop and mobile.
